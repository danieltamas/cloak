// threat model

AI agents don't hack you.
They just read your files.

Every coding agent scans your working directory for context. Your .env is just another file. .gitignore only protects git. It does nothing for local file reads.

01

Agents read .env on every prompt

Claude Code, Cursor, Copilot, and Codex index your project to build context. That includes every .env file with your Stripe live keys, database URLs, JWT secrets, private keys, and wallet mnemonics. No exploit required — it's how they work.

02

Secrets travel over the wire

File contents are sent to AI providers as part of API requests. Your production database password is now in a context window on someone else's infrastructure. It's in logs, in caches, potentially in fine-tuning pipelines.

03

One leak is permanent damage

A leaked sk_live_ key can drain Stripe accounts. A leaked database URL exposes user data. A leaked private key or wallet mnemonic in your .env means irreversible loss of funds. By the time you rotate, the window is open.

04

No existing tool solves this

.gitignore protects git, not disk. Vault/Infisical are for teams and production. Environment managers don't encrypt at rest. Nothing prevents a local process from reading cat .env and getting your real credentials. Until Cloak.

// protocol

Intercept. Encrypt. Substitute.

Real credentials go into an AES-256-GCM encrypted vault. Structurally valid fakes go to disk. Everything is local. No servers, no accounts, no network calls.

1

Initialize protection

Run cloak init. Cloak scans for secrets using pattern matching and entropy analysis. Real values are encrypted to a local vault. Sandbox replacements are written to disk. A recovery key and CLI access password are set.

$ cloak init
scanning .env...
[ok] 11 secrets detected
[ok] vault created (AES-256-GCM)
[ok] sandbox written to disk
[!!] recovery key: CLOAK-8f2a-b9c1-d4e5-f6a7-b8c9-d0e1
save this. it will not be shown again.
[ok] CLI password set
2

Edit transparently

The VS Code / Cursor extension intercepts file opens and saves. You see and edit real credentials in your buffer. On save, the extension encrypts to the vault and writes sandbox values to disk. Zero workflow change.

# your editor buffer (decrypted):
STRIPE_KEY=sk_live_4eC39HqLyjWDar...

# on disk after save (sandbox):
STRIPE_KEY=sk_test_cloak_sandbox_000000000000
3

Run with real values

cloak run decrypts the vault and injects real environment variables into any child process. Touch ID or password required — AI agents can't authenticate. No files touched.

$ cloak run npm start
[auth] Touch ID / password
# process.env.STRIPE_KEY =
sk_live_4eC39HqLyjWDar... (real)

$ cat .env | grep STRIPE
sk_test_cloak_sandbox_000000000000 (sandbox)
4

Recover anytime

Lost your keychain? Switched machines? cloak recover restores vault access from your recovery key. PBKDF2-SHA256 with 100k iterations protects the recovery file. No data is ever lost.

$ cloak recover
enter recovery key: CLOAK-8f2a-b9c1-...
[ok] recovery file decrypted
[ok] keychain key restored
[ok] vault accessible

// security model

Three layers deep

Encryption protects the vault. Biometric auth blocks the CLI. Modal gates block the extension. Every failure mode favors visibility over lockout.

OS keychain storage

Encryption keys in macOS Keychain, libsecret, or Windows Credential Manager. Protected by your OS login. Never on disk as plaintext.

Biometric auth gate

Touch ID on macOS, PBKDF2-hashed password on all platforms. CLI commands that reveal secrets require authentication. AI agents can't bypass.

Non-interactive rejection

No TTY means no access. Piped input, scripted access, and CI runners are blocked. Only interactive terminals can authenticate.

VS Code modal gate

Cloak Terminal with real env vars requires clicking a modal dialog. AI agents can't interact with VS Code modals.

Zero plaintext on disk

No backup files, no temp files that persist, no recovery dumps. Plaintext only exists in editor memory buffers.

Zero network calls

No telemetry, no analytics, no cloud sync. Only cloak update optionally checks GitHub releases. Everything else is offline.

Atomic file writes

Write-to-temp-then-rename. Interrupted saves can't corrupt vaults or leave half-written .env files.

Fail toward visibility

Decrypt failure on open shows sandbox with a warning. Encrypt failure on save writes real values to disk. Temporary leak beats permanent data loss.

Auditable source

MIT licensed. Rust CLI and TypeScript extension. Read every line. Build from source. No obfuscation, no binaries you can't inspect.
